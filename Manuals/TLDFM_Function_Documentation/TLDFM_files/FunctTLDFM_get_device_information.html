TLDFM_get_device_information

ViStatus TLDFM_get_device_information (ViSession instrumentHandle, ViUInt32 deviceIndex, ViChar _VI_FAR manufacturer[], ViChar _VI_FAR instrumentName[], ViChar _VI_FAR serialNumber[], ViPBoolean deviceAvailable, ViChar _VI_FAR resourceName[]);

Purpose

Get identification information of a connected device.
You don't have to open a session with the device with TLDFM_init() before you can use this function.

Parameters

Input
Name Type Description
instrumentHandle ViSession This parameter is needed for IVI compliance. Set to VI_NULL.
deviceIndex ViUInt32 The device's index. Valid values range from 0 to (number of connected devices - 1) (see TLDFM_get_device_count()).
Output
Name Type Description
manufacturer ViChar[] A TLDFM_BUFFER_SIZE (256) byte string buffer to receive the manufacturer name. You may pass VI_NULL if you don't need this value.
instrumentName ViChar[] A TLDFM_MAX_INSTR_NAME_LENGTH (28) byte string buffer to receive the instrument/model name. You may pass VI_NULL if you don't need this value.
serialNumber ViChar[] A TLDFM_MAX_SN_LENGTH (28) byte string buffer to receive the serial number. You may pass VI_NULL if you don't need this value.
deviceAvailable ViBoolean (passed by reference) Returns VI_TRUE if the device is available (not used by another program) or VI_FALSE.
You may pass VI_NULL if you don't need this value.
resourceName ViChar[] A VI_FIND_BUFLEN (256) byte string buffer to receive the ressource identification string. Use this string in function TLDFM_init().
You may pass VI_NULL if you don't need this value.

Return Value

Name Type Description
status ViStatus Operational return status. Contains either a completion code or an error code. Instrument driver specific codes that may be returned in addition to the VISA error codes defined in VPP-4.3 and vendor specific codes, are as follows.
Completition Codes
----------------------------------------------------------------
VI_SUCCESS              Initialization successful
Vendor Specific Codes
----------------------------------------------------------------
For error codes and descriptions see <Error Message>.
