TLDFMX_relax

ViStatus TLDFMX_relax (ViSession instrumentHandle, ViUInt32 devicePart, ViBoolean isFirstStep, ViBoolean reload, ViReal64 _VI_FAR relaxPatternMirror[], ViReal64 _VI_FAR relaxPatternArms[], ViPInt32 remainingSteps);

Purpose

This function calculates neccessary relax patterns for the specified piezo parts of the Deformable Mirror.
The function needs to be called repeatedly within a loop until the 'Remaining Steps' parameter returns 0.
Each returned segment or arm pattern have to be set separately to the physical device by using TLDFM base driver functions like TLDFM_set_segment_voltages, TLDFM_set_tilt_voltages or TLDFM_set_voltages.
The relax function will minimize piezo drift in order to get a more stable mirror surface.

Parameters

Input
Name Type Description
instrumentHandle ViSession This parameter accepts the Instrument Handle returned by the Initialize function to select the desired instrument driver session.
devicePart ViUInt32 This parameter determines the target part to be relaxed.
isFirstStep ViBoolean Set 'Is First Step' to 'VI_TRUE' for the first relax step and to  'VI_FALSE' for all following steps.
reload ViBoolean If VI_TRUE the function will calculate patterns symmetric to the actual voltage. The last step will return the voltage pattern prior relaxing again.
If VI_FALSE each piezo of the device part will be relaxed to 100V.
Note: The initial relax amplitude for each segment or tilt arm may be different because the maximum available symmetric voltage range depends on its actual voltage.
For instance, the initial relax amplitude of a segment having 150 V is 50V, allowing a symmetric relaxing between 100V and 200V.
Call function TLDFM_get_device_configuration to get the valid voltage range.
Output
Name Type Description
relaxPatternMirror ViReal64[] Returns the Mirror Pattern to be set by caller using base driver functions like TLDFM_set_segment_voltages.
If 'Arms' is selected as 'Device Part' only you may pass VI_NULL.
Note:
This pattern is already limited to the allowed voltage range.
relaxPatternArms ViReal64[] Returns the Arms Pattern to be set by caller using base driver functions like TLDFM_set_tilt_voltages.
If 'Mirror' is selected as 'Device Part' only you may pass VI_NULL.
Note:
This pattern is already limited to the allowed voltage range.
remainingSteps ViInt32 (passed by reference) Returns a number how often the function have to be called again.
Note: If remaining steps equals 0 the function don't have to be called again.

Return Value

Name Type Description
status ViStatus Operational return status. Contains either a completion code or an error code. Instrument driver specific codes that may be returned in addition to the VISA error codes defined in VPP-4.3 and vendor specific codes, are as follows.
Completition Codes
----------------------------------------------------------------
VI_SUCCESS              Close successful
Vendor Specific Codes
----------------------------------------------------------------
For error codes and descriptions see <Error Message>.
