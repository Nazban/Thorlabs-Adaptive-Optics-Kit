TLDFMX_convert_measured_zernike_amplitudes

ViStatus TLDFMX_convert_measured_zernike_amplitudes (ViSession instrumentHandle, ViReal32 _VI_FAR measuredZernikeAmplitudes[], ViReal64 _VI_FAR deviceZernikeAmplitudes[]);

Purpose

This function converts absolute Zernike amplitudes (in microns) measured by a Wavefront Sensor into normalized Zernike amplitudes used in the TLDFMX Zernike pattern generation.
This can be used for calculating control steps like it is done in function TLDFMX_get_flat_wavefront or for reverse-normalizing  the TLDFM Zernike amplitudes into microns.
In addition this function converts the data type from ViReal32 (float) to ViReal64 (double) and shifts the array positions. This is advantageous to adapt Zernike data retrieved from a Thorlabs Wavefront Sensor.
Note:
A successful run of function TLDFMX_measure_system_parameters is a precondition to use this function.

Parameters

Input
Name Type Description
instrumentHandle ViSession This parameter accepts the Instrument Handle returned by the Initialize function to select the desired instrument driver session.
measuredZernikeAmplitudes ViReal32[] This parameter accepts an array of absolute Zernike amplitudes measured by a Wavefront Sensor in microns.
The data type is ViReal32 (float) and meaningful indices range from [4] for Z4 to [15] for Z15.
Output
Name Type Description
deviceZernikeAmplitudes ViReal64[] This parameter returns normalized Zernike amplitudes (-1.0 to +1.0) related to the DMP40 device which can be used for TLDFMX Zernike pattern generation functions.
The data type is ViReal64 (double) and meaningful indices range from [0] for Z4 to [11] for Z15.
Note:
Although the measured amplitudes are mapped to a normalized -1.0 to +1.0 representation, the returned amplitudes may violate this range because there is no built-in range check.

Return Value

Name Type Description
status ViStatus Operational return status. Contains either a completion code or an error code. Instrument driver specific codes that may be returned in addition to the VISA error codes defined in VPP-4.3 and vendor specific codes, are as follows.
Completition Codes
----------------------------------------------------------------
VI_SUCCESS              Close successful
Vendor Specific Codes
----------------------------------------------------------------
For error codes and descriptions see <Error Message>.
