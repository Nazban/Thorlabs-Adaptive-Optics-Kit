TLDFMX_get_pattern_range_check

ViStatus TLDFMX_get_pattern_range_check (ViSession instrumentHandle, ViPBoolean enabled);

Purpose

This function returns the actual pattern range check setting.
When enabled, the caller can be sure that all segment or arm voltages returned by TLDFMX functions are within the allowed range and can be directly set to the device by base driver functions like TLDFM_set_segment_voltages, TLDFM_set_tilt_voltages or TLDFM_set_voltages.
Note: The range check is enabled by default.

Parameters

Input
Name Type Description
instrumentHandle ViSession This parameter accepts the Instrument Handle returned by the Initialize function to select the desired instrument driver session.
Output
Name Type Description
enabled ViBoolean (passed by reference) This parameter returns the actual setting of the pattern range check.
Available settings:
VI_TRUE   range check enabled (default)
VI_FALSE  range check disabled
Note:
When the voltage pattern range check is disabled, the caller is responsible for limiting the voltage pattern returned from TLDFMX functions to the allowed range before they can be set to the device by base driver functions like TLDFM_set_segment_voltages, TLDFM_set_tilt_voltages or TLDFM_set_voltages.

Return Value

Name Type Description
status ViStatus Operational return status. Contains either a completion code or an error code. Instrument driver specific codes that may be returned in addition to the VISA error codes defined in VPP-4.3 and vendor specific codes, are as follows.
Completition Codes
----------------------------------------------------------------
VI_SUCCESS              Get successful
Vendor Specific Codes
----------------------------------------------------------------
For error codes and descriptions see <Error Message>.
